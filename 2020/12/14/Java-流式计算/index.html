Consumer<T>
消费型接口		 T void 对类型为T的对象应用操作，包含方法：void accept(T t);
Supplier<T>
供给型接口		 无 T 返回类型为 T 的对象，包含方法：T get();
Function<T, R>
函数型接口		 T R 对类型为 T 的对象应用操作，并返回结果。结果是 R 类型的对象。包含方法：R apply(T t);
Predicate<T>
断定型接口		 T boolean 确定类型为 T 的对象是否满足某约束，并返回 boolean 值。包含方法：boolean test(T t);

Stream 流

流（Stream）是什么

是数据渠道，用于操作数据源（集合、数组等）所生成的元素序列。

集合讲的是数据，流讲的是计算！

特点

  • Stream 自己不会存储元素
  • Stream 不会改变源对象。相反，他们会返回一个持有结果的新 Stream
  • Stream 操作是延迟执行的。这意味着他们会等到需要结果的时候才执行

Stream 使用

  1. 创建一个 Stream：一个数据源（数组、集合）
  2. 中间操作：一个中间操作，处理数据源数据
  3. 终止操作：一个终止操作，执行中间操作链，产生结果
